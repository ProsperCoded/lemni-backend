### 3. DTO ✅
- `src/webhook/dto/webhook.dto.ts` — `NombaWebhookEventSchema` with typed `NombaWebhookEventDto`
- `event_type` is plain `z.string()` (not enum) to accept any Nomba event type
- Schema matches Nomba's payload shape exactly

### 4. Signature Verification ✅
- `src/webhook/webhook-signature.util.ts` — Two exported functions:
  - `buildNombaSigningString(payload, timestamp): string` — reconstructs 9-field colon-delimited string per Nomba docs
  - `verifyNombaSignature(payload, timestamp, signatureHeader, secret): boolean` — HMAC-SHA256 with timingSafeEqual
- Handles `responseCode` null coercion correctly
- Length pre-check prevents timing-attack on mismatched lengths

### 5. Service ✅
- `src/webhook/webhook.service.ts` — `@Injectable()` with `processNombaEvent(event: NombaWebhookEventDto)`
- Forward-only state machine: terminal transactions (`success`/`failed`) return `duplicate` without re-advancing
- `payment_success` → updates transaction, calls `advanceSubscription()` to set subscription `active` + new `currentPeriodEnd`
- `payment_failed` → updates transaction to `failed`, does not advance subscription
- Other event types (`payment_pending`, unknown) → returns `ignored`, no DB writes
- Orphaned nombaRef → logs as `orphaned_webhook`, returns `orphaned` status, no DB writes
- Imports `computeNextPeriodEnd` from shared util; no duplication

### 6. Controller ✅
- `src/webhook/webhook.controller.ts` — `@Controller('api/v1/webhooks')` with `@Post('nomba')`
- `@ApiTags('developer-apis/webhooks')` correctly tags the endpoint
- Schema validation happens **before** signature check, returns soft `200` on malformed (not `400`)
- Signature failure → `401 UnauthorizedException` with logged IP
- No auth guards — signature verification *is* the auth mechanism
- Logs validation failures and sig verification failures with client IP
- `@HttpCode(HttpStatus.OK)` explicitly sets 200 for non-error responses

### 7. Module ✅
- `src/webhook/webhook.module.ts` — `@Module({ controllers, providers })` with WebhookController and WebhookService
- `src/app.module.ts` — WebhookModule registered in `imports` array
- No explicit DatabaseModule import needed (global DRIZZLE_PROVIDER is available)

### 8. OpenAPI Reconciliation ✅
- `pnpm lint:openapi` detected drift (endpoint missing from spec)
- `pnpm lint:openapi -- --write` auto-synced the spec
- `docs/openapi/openapi.yaml` now contains:
  ```yaml
  /api/v1/webhooks/nomba:
    post:
      description: Verifies the Nomba signature and updates the matching transaction/subscription state accordingly.
      operationId: WebhookController_handleNombaWebhook
      responses:
        '200': Event processed, ignored, orphaned, or deduplicated

        - developer-apis/webhooks
  ```
