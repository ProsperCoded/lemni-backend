**Status:** DRAFT (Awaiting Review)

**Scope:** Support transaction history retrieval and filtering for the merchant dashboard (`BillingModule`).

---

## 1. Goal
Add a secure admin endpoint `GET /admin/transactions` that allows authenticated merchants to view, paginate, and filter transaction records.

---

## 2. Requirement Delta

### REQUIREMENTS.md Updates
Add a new task under **4. BILLING & DOMAIN LOGIC (BillingModule)**:
- [ ] **4.5. Transaction History & Filtering**
  - Implement `GET /admin/transactions` for retrieving merchant's transactions.
  - Support filters: `status`, `customerId`, `subscriptionId`, `startDate`, `endDate`.
  - Support pagination: `limit` (default 20), `offset` (default 0).
  - Ensure results are scoped strictly to the authenticated merchant (`merchantId` from JWT).

## 3. Live Code Facts & Proposed Changes

### 3.1. Database Schema
Update `transactions` table in [schema.ts](file:///home/proseper/workspace/Collaborations/lemni-backend/src/database/schema.ts#L74-L85) to include `merchantId` and `customerId` columns for direct querying:
```typescript
export const transactions = sqliteTable('transactions', {
  id: text('id').primaryKey(),
  merchantId: text('merchant_id')
    .notNull()
    .references(() => merchants.id, { onDelete: 'cascade' }),
  customerId: text('customer_id')
    .notNull()
    .references(() => customers.id, { onDelete: 'cascade' }),
  subscriptionId: text('subscription_id').references(() => subscriptions.id, {
    onDelete: 'set null',
  }),
  amount: real('amount').notNull(),
  status: text('status', { enum: ['pending', 'success', 'failed'] }).notNull(),
  nombaRef: text('nomba_ref'),
  payload: text('payload'),
  response: text('response'),
  createdAt: text('created_at').default(sql`CURRENT_TIMESTAMP`),
});
```
