# RF-002: Password Reset Endpoint Implementation Plan

**Status:** COMPLETE ✅

**Date:** 2026-07-04

**Scope:** Support password reset verified by current password in the `AuthModule`.

---

## 1. Goal
Provide secure, self-service password reset capabilities for merchant dashboard accounts by verifying the user's current password.

---

## 2. Requirement Delta

### REQUIREMENTS.md Updates
Added task under **2. SECURITY & AUTHENTICATION (AuthModule)**:
- [x] **2.5. Password Reset Flow**
  - Implement `POST /auth/reset-password` to reset password by verifying old password.
  - **Unhappy paths:**
    - Request reset for non-existent or invalid email → Return `400 Bad Request` with `Invalid email or password`.
    - Incorrect old password → Return `400 Bad Request` with `Invalid email or password`.
    - New password too short (less than 8 characters) → Return `400 Bad Request`.

### Feature File Updates (`features/FR-002_security_authentication.feature`)
Added Scenario:
```gherkin
  Scenario: Complete Password Reset with Old Password Verification
    Given a merchant has registered with email "admin@example.com" and password "OldSecurePassword123"
    When they POST to "/auth/reset-password" with email "admin@example.com", oldPassword "OldSecurePassword123", and newPassword "NewSecurePassword123"
    Then the password is successfully updated in the database
```

---

## 3. Live Code Facts & Proposed Changes

### 3.1. Database Schema
No custom schema tokens or reset columns were added (kept schema stateless as verification relies purely on comparing existing credentials).

### 3.2. Auth Service (`src/auth/auth.service.ts`)
Added:
- `resetPassword(email: string, oldPassword: string, newPassword: string): Promise<{ success: boolean }>`
  - Look up merchant by email.
  - Verify old password matches current `hashedPassword` using `bcrypt.compare`.
  - Validate new password is >= 8 characters.
  - Hash new password using `bcrypt` and update the database record.

### 3.3. Auth Controller (`src/auth/auth.controller.ts`)
Added route:
- `POST /auth/reset-password` (Public)
  - Inputs: `email` (string), `oldPassword` (string), `newPassword` (string)
  - Tagged under: `merchant-dashboard/auth` (inherits from controller decorator `@ApiTags('merchant-dashboard/auth')`)
